# 커밋 컨벤션 가이드  

본 문서는 기술 면접 준비 가이드의 품질 향상과 효율적인 협업을 위해 통일된 커밋 메시지 규칙을 안내합니다.  
여러분의 소중한 기여가 보다 명확하고 체계적으로 관리될 수 있도록 아래 규칙에 맞는 커밋 메시지를 작성해 주세요.  



## 커밋 메시지 작성 형식  


- **태그**: 아래 표에 정의된 커밋 유형 중 하나를 사용합니다.  
- **변경 내용 요약**: 한눈에 어떤 변경 사항이 있는지 알 수 있도록 간결하고 명확하게 작성합니다.  



## 커밋 태그 목록  

| 태그      | 설명                          | 예시                                 |
|------------|------------------------------|---------------------------------------|
| `ADD`      | 새로운 질문이나 내용을 추가  | ADD :: 네트워크 섹션에 DNS 질문 추가  |
| `FIX`      | 오타, 잘못된 정보 수정       | FIX :: 알고리즘 설명 오타 수정        |
| `UPDATE`   | 기존 질문/답변 보완 또는 개선 | UPDATE :: 데이터베이스 인덱스 설명 보완 |
| `REMOVE`   | 불필요한 질문/내용 삭제      | REMOVE :: 중복된 운영체제 질문 제거   |
| `REFACTOR` | 문서 구조나 표현 방식 개선 (내용 변경 없음) | REFACTOR :: 목차 구조 정리         |
| `DOCS`     | 문서 전반적인 수정 및 보완    | DOCS :: 기여 방법 섹션 추가            |

---

### 📖 커밋 작성 시 유의사항  
   - 어떤 내용을 추가/수정/삭제했는지 구체적으로 작성해 주세요.  
   - 예) `ADD :: 자료구조 섹션에 힙 정렬 질문 추가` (O)  
         `ADD :: 내용 추가` (X)  
  
   - 여러 개의 태그가 필요한 경우, 커밋을 분리하는 것이 좋습니다.  
   - 오타 수정, 문장 표현 개선, 목차 추가 등 사소한 수정도 큰 도움이 됩니다.  

---

여러분의 작은 기여가 이 문서를 더욱 가치 있게 만듭니다.  
편하게 참여해 주세요! 🚀
